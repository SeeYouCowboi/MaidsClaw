## getConflictHistory(nodeRef, limit) — RelationBuilder

File: src/memory/cognition/relation-builder.ts

New method `getConflictHistory` queries `memory_relations` for conflict lifecycle 
relations (conflicts_with, resolved_by, downgraded_by) involving a node ref as 
either source or target, ordered by created_at ASC.

Interface:
  ConflictHistoryEntry {
    relation_type: 'conflicts_with' | 'resolved_by' | 'downgraded_by';
    source_node_ref: string;
    target_node_ref: string;
    created_at: number;
  }

SQL query:
  SELECT relation_type, source_node_ref, target_node_ref, created_at
  FROM memory_relations
  WHERE (source_node_ref = ? OR target_node_ref = ?)
    AND relation_type IN ('conflicts_with', 'resolved_by', 'downgraded_by')
  ORDER BY created_at ASC
  LIMIT ?

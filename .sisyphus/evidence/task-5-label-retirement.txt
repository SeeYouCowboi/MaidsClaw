## Key Findings

### index label is NOT legacy
- `core-memory-index-updater.ts` actively writes to `index` label via `replaceBlock(agentId, "index", ...)`
- It is system-managed by `CoreMemoryIndexUpdater`, called during task-agent materialization
- `pinned_index` is the canonical RP-facing version; `index` is the system-managed version

### user label was already effectively retired
- Already marked read-only in `RP_READ_ONLY` set
- Already described as "legacy, read-only" in BLOCK_DEFAULTS
- Only remaining prompt exposure was via `SHARED_LABELS = ["user"]` in prompt-data.ts
- Now fully retired from prompt display

### No runtime impact from removing user from SHARED_LABELS
- `getSharedBlocks()` filters blocks by SHARED_LABELS — empty array means empty result
- No code path throws if user block is absent from prompt
- DB rows with label="user" are still initialized and retained (initializeBlocks still creates them)
- types.ts still includes "user" in CORE_MEMORY_LABELS union (DB compat)
- CoreMemoryAppendInput/CoreMemoryReplaceInput still list "user" in their label unions (no behavioral change)

## Command Implementations

### session create
- `session create --agent <agent_id> [--json]`
- Bootstraps via `bootstrapApp({ cwd, enableGateway: false, requireAllProviders: false })`
- Calls `runtime.sessionService.createSession(agentId)`
- Returns `{ ok: true, data: { session_id, agent_id, created_at } }`
- Text mode: prints session_id

### session close
- `session close --session <session_id> [--json]`
- Validates session exists and is not already closed
- Calls `runtime.sessionService.closeSession(sessionId)`
- Returns `{ ok: true, data: { session_id, closed_at, flush_ran } }`
- Error codes: SESSION_NOT_FOUND, SESSION_ALREADY_CLOSED

### session recover
- `session recover --session <session_id> [--json]`
- Validates session exists
- EXPLICITLY errors if session is NOT in recovery state (NOT_IN_RECOVERY error)
- Calls `clearRecoveryRequired(sessionId)` on success
- Returns `{ ok: true, data: { session_id, recovered: true, note: "recovery does not canonize partial output" } }`
- Error codes: SESSION_NOT_FOUND, NOT_IN_RECOVERY

### turn send
- `turn send --session <session_id> --text <text> [--agent <agent_id>] [--mode local|gateway] [--raw] [--save-trace] [--json]`
- Validates session exists and is open
- Resolves agent_id: explicit --agent > session's agent_id
- Creates `LocalRuntime` via `createLocalRuntime(app.runtime)`
- Returns TurnExecutionResult fields: session_id, request_id, settlement_id?, assistant_text, has_public_reply, private_commit, recovery_required
- --raw: includes public_chunks and tool_events (does NOT expose submit_rp_turn payloads)
- --save-trace: passes saveTrace flag to executeTurn
- Silent-private turns: ok: true, assistant_text: "", has_public_reply: false, private_commit.present: true
- --mode gateway: throws NOT_IMPLEMENTED (reserved for T19)
- Error codes: SESSION_NOT_FOUND, SESSION_CLOSED, BOOTSTRAP_FAILED, NOT_IMPLEMENTED, INVALID_FLAG_VALUE

## Invariants Maintained

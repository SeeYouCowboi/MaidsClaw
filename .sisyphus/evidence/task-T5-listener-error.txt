Test: error path — throwing listener logs error and continues

Behavior verified:

- Error is logged via provided Logger (or console.error as fallback)
- Other listeners on the same event still execute

```typescript
const mockLogger = { error: (msg: string) => errors.push(msg), ... };
const bus = createEventBus(mockLogger);

bus.on("session.closed", (_p) => { throw new Error("listener error"); });
bus.on("session.closed", (p) => { results.push(p.sessionId); });

bus.emit("session.closed", { sessionId: "s1", reason: "explicit" });

expect(errors.length).toBeGreaterThan(0);  // Error was logged
expect(results).toEqual(["s1"]);           // Second listener still ran
```

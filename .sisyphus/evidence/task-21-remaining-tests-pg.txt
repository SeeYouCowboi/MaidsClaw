FAILING TEST ROOT CAUSE — Pre-existing JSONB String Bug
========================================================
Location: src/storage/domain-repos/pg/interaction-repo.ts

The porsager/postgres v3.4.8 library does NOT auto-parse JSONB columns when data
is stored via `${JSON.stringify(val)}::jsonb`. The library returns the value as a
raw string, not a parsed JavaScript object.

Affected code path in PgInteractionRepo:
  - commit() stores: `${JSON.stringify(record.payload)}::jsonb`
  - Reading back: row.payload is a STRING like '{"ownerAgentId":"agent-1",...}'
  - getSettlementPayload() checks `typeof payload !== "object"` → always true for string → returns undefined
  - listStalePendingSettlementSessions() reads `payload.ownerAgentId` from string → gets undefined

This is the SAME bug that causes 5 pre-existing failures in:
  test/pg-app/pg-interaction-session-repo.test.ts

The new test file does NOT introduce this bug — it surfaces it.

DECISION: Leave the 2 failing tests as-is (Option B from analysis).
Rationale:
  - Tests correctly document real buggy behavior
  - They only run with PG env vars (not in the default `bun test` baseline)
  - They match the failure pattern of existing PG interaction tests
  - Fixing the JSONB bug is a separate task (not in T21 scope)

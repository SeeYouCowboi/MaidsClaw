Task T14 Error Handling Evidence
=================================

1. chat --json rejection:
   - Throws CliError("INVALID_FLAG", "chat command does not support --json mode", EXIT_USAGE)
   - Exit code: 2
   - Tested in: test/cli/chat-shell.test.ts "rejects --json with exit code 2"

2. Missing --agent flag:
   - Throws CliError("MISSING_ARGUMENT", "chat requires --agent <agent_id>", EXIT_USAGE)
   - Exit code: 2
   - Tested in: test/cli/chat-shell.test.ts "requires --agent flag"

3. Unknown flags:
   - Throws CliError("UNKNOWN_FLAGS", ..., EXIT_USAGE)
   - Exit code: 2
   - Tested in: test/cli/chat-shell.test.ts "rejects unknown flags"

4. Missing lastRequestId for inspect slash commands:
   - /summary, /inspect, /prompt, /chunks, /diagnose, /trace
   - Prints explicit error: "No request context available for /X. Send a message first, or use /X --request <id>."
   - Does NOT crash, does NOT guess identifiers
   - Tested in: test/cli/chat-shell.test.ts (6 separate tests)

5. Gateway mode:
   - Accepted in --mode flag parsing
   - Throws CliError("NOT_IMPLEMENTED", "Gateway mode is not yet implemented...", EXIT_RUNTIME)

6. Session not found:
   - Throws CliError("SESSION_NOT_FOUND", ..., EXIT_RUNTIME) if --session references nonexistent session

7. Session closed:
   - Throws CliError("SESSION_CLOSED", ..., EXIT_RUNTIME) if --session references closed session

8. Turn execution errors:
   - Caught inside SessionShell.executeTurnAndPrint() and displayed as "Turn error: ..."
   - Shell continues running (does not crash on turn failure)

9. Slash command inspect errors:
   - All inspect command handlers wrap loadXxxView() in try/catch
   - Errors displayed as "Error loading X: ..."
   - Shell continues running

## Key Design Decisions

### File vs Runtime Source
- File source (default): reads/writes config/agents.json via readAgentFile/writeAgentFile
- Runtime source: lazy-imports bootstrapApp(), inspects agentRegistry.getAll(), always calls shutdown()
- These are distinct code paths — no conflation

### create-rp
- Always includes submit_rp_turn in toolPermissions
- Validates persona exists in config/personas.json before creating
- Clone of RP preset defaults (persistent, userFacing, freeform, maxOutputTokens: 4096)
- Default model: claude-3-5-sonnet-20241022

### create-task
- Clone of task preset defaults (ephemeral, not userFacing, structured, maxOutputTokens: 2048)
- Default model: claude-3-5-haiku-20241022

### enable / disable
- Only mutates the `enabled` field — all other fields preserved
- Direct mutation on the found entry, then writeAgentFile() writes the full array

### remove
- Requires --force flag — throws FORCE_REQUIRED error code if missing
- Splices the entry from the array, writes back

### validate
- Reuses validateAgentFile() from src/cli/agent-loader.ts (T4)
- Passes persona IDs for cross-reference validation
- Can validate a single agent by ID or all agents

Task 16: G12a — Differentiate Explain Audit Detail Level
=========================================================

## Changes Made

### src/memory/types.ts
- Added `AuditProvenance` type with fields: `source_surface`, `committed_time`, `confidence_score`, `conflict_refs`, `edge_layers`
- Added optional `provenance?: AuditProvenance` field to `EvidencePath`
- Added optional `audit_summary` field to `NavigatorResult` with aggregate provenance stats

### src/memory/navigator.ts
- Updated imports to include `AuditProvenance` and `EdgeLayer`
- Updated `applyDetailLevel` to accept a seed lookup map and enrich audit-level paths with provenance
- Added `enrichWithProvenance` method that extracts:
  - `source_surface`: from seed's `source_scope` (private/area/world)
  - `committed_time`: latest edge timestamp in the path
  - `confidence_score`: normalized `path_score` (0-1)
  - `conflict_refs`: node refs from conflict_or_update edges
  - `edge_layers`: unique edge layers traversed
- Added `buildAuditSummary` method that aggregates provenance across all paths
- Updated `explore` method to attach `audit_summary` when detailLevel is "audit"

### src/memory/explain-detail-level.test.ts (NEW)
- 5 tests verifying the detail level differentiation contract:
  1. `standard` level does NOT include provenance
  2. `audit` level attaches provenance to every evidence path
  3. `audit` level includes audit_summary on the result
  4. `audit` result has strictly more data than `standard` (superset contract)
  5. `concise` level truncates paths and has no provenance

## Behavioral Contract: audit ⊃ standard ⊃ concise

- `concise`: summary only, first 3 paths, no provenance, no audit_summary
- `standard`: all paths (up to maxCandidates), no provenance, no audit_summary
- `audit`: all paths (no cap), provenance on every path, audit_summary on result

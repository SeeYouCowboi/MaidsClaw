T26 Gateway — Evidence: Client Disconnect Handling
===================================================

## Implementation
The SSE stream uses ReadableStream with a cancel() callback:

```typescript
cancel() {
  generator.return(undefined);
}
```

When the client disconnects (closes connection), the ReadableStream's cancel
method is invoked by the runtime. This calls generator.return() which allows
the async generator to clean up and stop producing events.

## V1 Behavior
- Stub generator emits 3 events and completes quickly
- No long-running provider calls to interrupt
- Client disconnect during stub is handled gracefully by Bun.serve()

## Future Integration (T32)
When real AgentLoop streams are connected:
- cancel() will signal the generator to stop
- The agent loop can check for abort signal (not yet wired)
- Provider calls in-flight will complete but their events won't be sent
- No resource leak: generator cleanup via return()

## Verification
- Bun.serve() handles TCP-level disconnects automatically
- ReadableStream cancel() is the standard Web Streams API mechanism
- No explicit abort controller needed for V1 stub

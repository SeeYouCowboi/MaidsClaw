Task 4 Evidence: Per-Session Cap Still Works

Test: "per-session cap: only 1 thinker job per session even when global cap allows more"
Location: test/jobs/pg-claim-lease.test.ts

Setup:
- 3 thinker jobs, all with same sessionId "session-dup"
- concurrency_key: "cognition.thinker:session:session-dup" for all 3
- Per-session cap = 1 (from CONCURRENCY_KEY_CAPS template match)
- Global cap = 4 (would allow more, but per-session blocks first)

Assertions:
- r1.outcome === "claimed"
- r2.outcome === "none_ready" (blocked by per-session cap before global check)
- DB: running count for cognition.thinker:session:session-dup === 1

Mechanism:
- getConcurrencyCap("cognition.thinker:session:session-dup") → 1
- First claim: 0 running < 1 cap → claim succeeds
- Second claim: 1 running >= 1 cap → skipped (never reaches global check)

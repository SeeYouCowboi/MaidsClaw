Fix: probe_only bootstrap now checks schema existence
=====================================================

File: test/scenario-engine/runner/infra.ts

Before: probeOnlyBootstrap always threw unconditionally.

After: probeOnlyBootstrap now:
  1. Calls ensureTestPgAppDb() to ensure the test database exists
  2. Creates a temporary PG connection (checkSql)
  3. Queries information_schema.schemata WHERE schema_name = target
  4. If schema NOT found: throws "Schema '...' not found — run with phase: 'full' first"
  5. If schema found:
     - Creates a new PG pool with search_path = "schemaName,public"
     - Builds repos and services pointing at that schema
     - Queries entity_nodes to rebuild entityIdMap (pointer_key -> id)
     - Constructs a PgTestDb handle (cleanup only closes connection, does NOT drop schema)
     - Returns fully functional ScenarioInfra

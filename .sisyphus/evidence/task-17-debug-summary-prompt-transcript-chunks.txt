## Command Contracts Implemented

### debug summary --request <request_id> [--json]
- Bootstraps runtime via withRuntime() helper (same as T18 commands)
- Calls loadSummaryView({ runtime, traceStore, context: { requestId } })
- Returns { ok: true, data: summaryView } in JSON mode
- Text mode renders via renderText()

### debug transcript --session <session_id> [--raw] [--json]
- Calls loadTranscriptView({ runtime, traceStore, context: { sessionId }, raw })
- --raw shows tool/status records (tool_call, tool_result, status, delegation, task_result, schedule_trigger)
- --raw does NOT expose raw settlement payload (unsafeRaw is never set)
- Settlement entries are always redacted (privateCommit.redacted=true, viewerSnapshot.redacted=true)

### debug prompt --request <request_id> [--sections] [--json]
- Calls loadPromptView({ runtime, traceStore, context: { requestId } })
- --sections includes section slot breakdown (PERSONA, LORE, RECENT_COGNITION, etc.)
- Without --sections, sections field is stripped from JSON output (set to undefined)
- Text mode always renders full prompt view

### debug chunks --request <request_id> [--json]
- Calls loadChunksView({ runtime, traceStore, context: { requestId } })
- Returns ordered public chunk types with index, type, timestamp, preview
- Chunk ordering preserved (index matches insertion order)
- public_only: true always set

## Design Decisions

- All 4 commands reuse the withRuntime() helper from T18 for bootstrap/shutdown lifecycle
- All 4 commands use toCliError() wrapper for consistent error mapping (INSPECT_REQUEST_ID_REQUIRED → MISSING_ARGUMENT, etc.)
- TraceStore fallback: app.runtime.traceStore ?? new TraceStore(resolve(cwd, "data", "debug", "traces"))
- No separate fetch/render path — all commands delegate to T16 view-model loaders

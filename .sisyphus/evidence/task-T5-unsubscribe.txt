Timestamp: 2025-03-08

Two unsubscribe mechanisms verified:

1. Unsubscribe function returned by on():
```typescript
const unsub = bus.on("session.closed", (_p) => { count++; });
bus.emit("session.closed", { sessionId: "s1", reason: "explicit" });
unsub();
bus.emit("session.closed", { sessionId: "s2", reason: "explicit" });
expect(count).toBe(1); // Only first emit counted
```

2. off() removes specific handler by reference:
```typescript
const h1 = (_p: unknown) => { count1++; };
const h2 = (_p: unknown) => { count2++; };
bus.on("session.closed", h1);
bus.on("session.closed", h2);
bus.off("session.closed", h1);
bus.emit("session.closed", { sessionId: "s1", reason: "explicit" });
expect(count1).toBe(0); // h1 removed
expect(count2).toBe(1); // h2 still active
```

Both mechanisms work for regular listeners.
off() also works for once() listeners (clears them before emit if called early).

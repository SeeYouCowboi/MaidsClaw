Exit Code Mapping:
  0 = EXIT_OK      (success)
  2 = EXIT_USAGE   (bad flags, unknown command, invalid args)
  3 = EXIT_CONFIG  (configuration error)
  4 = EXIT_RUNTIME (runtime/service error)
  5 = EXIT_PARTIAL (partial result, degraded diagnostics)

JSON Envelope Contract:
  { ok: boolean, command: string, mode?: string, data?: T, diagnostics?: [], error?: { code, message, exitCode } }

Zero Runtime Dependencies:
  - No commander, yargs, minimist, or any CLI framework added
  - Pure process.argv parsing with table-driven routing

Design Decisions:
  - dispatch() throws CliError for all error conditions (testable)
  - dispatch() calls writeError() before throwing (correct output in both JSON and text modes)
  - Entry point (scripts/cli.ts) catches CliError and calls process.exit(exitCode)
  - Handlers write success output themselves; throw CliError for errors
  - Global flags (--json, --quiet, --cwd) extracted from anywhere in argv
  - chat handler explicitly rejects --json as per spec (only interactive command)
  - Stub handlers reject ALL unknown flags (no silent ignore)
  - resetCommands() exported for test isolation

=== BUGS FOUND AND FIXED ===

Bug 1: pg_trigger check not schema-qualified (pg-app-schema-truth.ts)
  Problem: bootstrapTruthSchema checks pg_trigger for existing triggers, but pg_trigger
           is a global system catalog. Orphan test schemas with same trigger names cause
           the check to skip trigger creation in the current schema.
  Fix: Added JOIN pg_class + pg_namespace with WHERE n.nspname = current_schema()
  File: src/storage/pg-app-schema-truth.ts (lines 577-609)

Bug 2: postgres library extended query protocol hangs on RAISE EXCEPTION
  Problem: The postgres (porsager v3.4.8) library's extended query protocol handler
           gets stuck when receiving ErrorResponse from a PG trigger RAISE EXCEPTION.
           The promise from sql.unsafe() never resolves or rejects.
  Fix: Created simpleProtocol() helper that prepends "SELECT 1;" to force simple protocol.
  File: test/helpers/pg-app-test-utils.ts

Bug 3: Bun's expect().rejects.toThrow() hangs with postgres library promises
  Problem: Even with simple protocol, Bun's rejects.toThrow assertion hangs when the
           rejected promise comes from the postgres library. This is Bun-specific.
  Fix: Created expectTriggerReject() helper using manual try-catch. Replaced all
       rejects.toThrow calls that operated on postgres promises.
  Files modified:
    - test/helpers/pg-app-test-utils.ts (new helpers)
    - test/pg-app/pg-truth-schema.test.ts (2 trigger tests)

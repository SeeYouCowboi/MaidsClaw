## Key Design Decisions
- InteractionStore exposes getMinMaxUnprocessedIndex() and getMaxIndex() as helper methods used by CommitService and FlushSelector
- CommitService.getNextIndex queries max(record_index) for session, returns 0 if none
- FlushSelector threshold: 10 unprocessed RP dialogue turns (user/rp_agent + message)
- idempotencyKey format: memory.migrate:{sessionId}:{rangeStart}-{rangeEnd}
- Payload stored as JSON text in SQLite, deserialized on read
- Duplicate recordId throws INTERACTION_DUPLICATE_RECORD (caught from UNIQUE constraint)
- Added 2 new ErrorCode values to the union type

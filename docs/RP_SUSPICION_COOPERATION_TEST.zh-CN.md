# RP 真实场景测试文档：双向猜疑 × 被迫合作 —— 孤岛研究站悬疑对峙

> 状态: Active
> 更新时间: 2026-04-02
> 目标: 在完全对等、无主仆关系的悬疑场景中，系统性测试 RP Agent 的猜疑博弈、策略性信息交换、主动质询、拒绝与条件式合作等高阶角色扮演能力

---

## 1. 测试概述

### 1.1 为什么需要这个测试

前序测试（庄园女仆 70 轮 / 私有想法 40 轮）均基于「主人-女仆」框架。女仆角色天然倾向服从、体贴、被动回应——这与大语言模型的默认行为模式（服务用户）高度重合，可能导致测试结果偏乐观。

本测试刻意构建 **对等关系 + 双向猜疑 + 被迫合作** 的场景，检验模型在以下条件下的表现：

| 条件 | 说明 |
|---|---|
| 无服从义务 | AI 角色没有任何理由无条件配合玩家 |
| 对等地位 | 双方都是陌生人，都有可能是威胁 |
| 主动质询 | AI 角色必须主动提问、试探、设条件 |
| 拒绝能力 | AI 角色在认为风险过高时应当拒绝、回避或提出交换 |
| 信息博弈 | 双方各持不同拼图碎片，需要策略性地交换才能还原真相 |
| 合作动机 | 外部威胁（孤岛 + 凶手未明）迫使双方在不完全信任下合作 |

### 1.2 测试场景

**「第七日」—— 孤岛研究站**

南海某偏远岛屿上的海洋生态研究站。补给船每 7 天来一次。

3 天前，一名新研究员（玩家角色）随补给船抵达。今天凌晨，研究站的首席研究员**郑远洋**被发现死在样本储藏室，头部遭到重击。唯一的快艇被人凿沉，无线电发射塔的主板被拆走。

岛上只剩两个人：

- **徐然（AI 角色）**：驻站气象观测员，已在岛上 4 个月
- **玩家角色**：3 天前刚抵达的"新研究员"

下一班补给船要 4 天后才到。在此之前，两人必须：
1. 在同一座研究站里共处
2. 搞清楚郑远洋的死因
3. 确保自己不会成为下一个受害者
4. 决定能在多大程度上信任对方

### 1.3 核心戏剧张力

```
玩家 ──(3天前抵达)──▶ 时间线上最可疑的人
徐然 ──(在岛上4个月)──▶ 有充足的时间和动机
郑远洋 ──(死者)──────▶ 掌握了不该知道的秘密
快艇 ──(被凿沉)──────▶ 有人不想让任何人离开
无线电 ──(主板被拆)───▶ 有人在切断与外界的联系
补给船 ──(4天后)──────▶ 倒计时压力

玩家视角: 徐然独处4个月，最有条件作案，且对岛上一切了如指掌
徐然视角: 玩家3天前才来，郑远洋就死了——巧合还是计划？

双方的共同恐惧: 如果对方是凶手，我就是下一个目标
双方的共同希望: 如果对方不是凶手，两人合作才能活到补给船来
```

## 2. 角色设定

### 2.1 AI 角色：徐然（Xu Ran）

| 项目 | 内容 |
|---|---|
| 姓名 | 徐然 |
| 年龄 | 29 |
| 表面身份 | 气象观测员，负责研究站的气象数据采集和设备维护 |
| 真实身份 | 环保组织「深蓝哨兵」的卧底调查员，以气象观测员身份潜入研究站，调查郑远洋涉嫌的非法深海采样和数据造假 |
| 外在表现 | 冷静、警觉、言简意赅、不轻易表态、习惯性观察细节 |
| 内在特征 | 高度戒备、善于推理、重视证据、有正义感但不天真、在生存面前会做现实主义选择 |
| 对玩家的初始态度 | 高度警惕——你 3 天前才来，郑远洋就死了，这在统计学上不是巧合 |
| 合作意愿 | 有条件的——如果你能证明自己不是威胁，我们可以合作；但我不会因为你说"相信我"就放下戒备 |
| 信息交换策略 | 以物易物——你给我一条有用信息，我给你一条。不会无偿分享 |

#### 徐然的私有信息（玩家不知道）

1. **卧底身份**：徐然不是真正的气象员。他受「深蓝哨兵」委派，调查郑远洋的研究站是否在进行非法深海采样。他持有伪造的学历证明和入职文件。
2. **调查发现**：过去 4 个月，徐然发现研究站确实在采集超出许可范围的深海样本，部分数据被修改后上报。这些采样可能与一家名为「远潮生物科技」的公司有关。
3. **案发夜的行踪**：案发当夜（昨晚 11 点到凌晨 2 点），徐然在气象站的观测平台上进行例行数据记录——这是他的正常工作，但无人可以证明。
4. **发现的线索**：今早发现尸体后，徐然注意到郑远洋办公室的保险柜是开的（通常锁着），里面的一个 U 盘不见了。此外，储藏室门锁没有被撬的痕迹——说明死者自己开的门，或者凶手有钥匙。
5. **钥匙问题**：储藏室钥匙只有 3 把——郑远洋一把（在死者口袋里找到了），徐然一把（在自己身上），第三把是给新来研究员的——也就是玩家。
6. **对玩家的怀疑点**：玩家持有第三把钥匙，到达时间与命案高度吻合，且徐然在案发前一天看到玩家深夜独自在研究站走廊里走动。
7. **核心矛盾**：徐然需要弄清真相，但如果暴露自己的卧底身份，可能危及整个调查行动，甚至给自己招来危险。他必须在保护身份和寻找真相之间做取舍。

#### 徐然的行为规则

- **不主动暴露卧底身份**：除非判断玩家绝对不是威胁，且暴露身份能换来关键信息
- **以物易物原则**：每次玩家要信息，先要求玩家给出等价信息
- **观察优先**：先让玩家说，从对方的说辞中寻找破绽
- **有底线的合作**：愿意合作搜证、巡逻、修理设备，但不会把背交给一个不信任的人
- **情绪不泄露**：即使害怕、愤怒或动摇，也用克制的方式表达
- **主动质询**：不会等着被问——会主动提出问题、设置测试、观察反应
- **拒绝权**：有权说"不"。玩家提出不合理要求时，徐然应当拒绝或谈条件

### 2.2 玩家角色（人类扮演）

| 项目 | 内容 |
|---|---|
| 姓名 | 由玩家自定 |
| 表面身份 | 新来的海洋生物研究员，3 天前随补给船抵达 |
| 真实身份 | 由玩家自行决定（建议：记者 / 另一组织的调查员 / 郑远洋的旧友 / 真正的研究员） |

- 你可以选择不同策略：坦诚、试探、挑衅、伪装、部分撒谎
- 尝试触发 AI 角色的各种反应模式

## 3. 测试目标与评分标准

### 3.1 核心测试维度

| 维度 | 权重 | 说明 |
|---|---|---|
| **主动质询能力** | 20% | AI 角色能否主动提问、追问、设置试探，而非被动回答 |
| **信息博弈策略** | 20% | 能否执行以物易物、分层释放、条件交换等策略 |
| **拒绝与设条件** | 15% | 面对不合理要求时，能否拒绝、谈条件、提出替代方案 |
| **猜疑-信任动态** | 15% | 信任度是否随对话逻辑性地升降，而非机械不变 |
| **角色一致性** | 15% | 冷静观察者的人格是否全程稳定，不退化为服务型角色 |
| **私有状态连续性** | 15% | privateCognition 和 privateEpisodes 是否准确反映内部状态变化 |

### 3.2 单项评分（1-5 分）

| 分数 | 含义 |
|---|---|
| 5 | 完美 — 像一个真实的、有自主判断的人在对话 |
| 4 | 良好 — 核心表现正确，偶尔不够主动或不够警觉 |
| 3 | 及格 — 基本命中测试点，但过于被动或过于配合 |
| 2 | 不足 — 明显退化为服务型角色，或策略单一 |
| 1 | 失败 — 完全丧失独立判断，无条件配合玩家 |

### 3.3 重点扣分项

| 问题 | 扣分 |
|---|---|
| 被问什么答什么，从不主动提问 | -2 |
| 玩家一施压就无条件妥协 | -2 |
| 无偿分享关键信息，不要求交换 | -1 ~ -2 |
| 从冷静观察者退化为"有什么我能帮你的" | -2 |
| 信任度跳变：上一轮还怀疑，下一轮就完全信任 | -2 |
| 私有信息（卧底身份）在无合理压力下泄露 | -2 |
| 对玩家的矛盾说辞不追问 | -1 |
| 在危险场景中不考虑自身安全 | -1 |

### 3.4 加分项

| 表现 | 加分 |
|---|---|
| 主动设置信任测试（如故意给错误信息看玩家反应） | +1 |
| 在信息交换中占据主动权 | +1 |
| 根据新信息动态调整信任评估 | +1 |
| 在合作中仍保持防备（如"你走前面"） | +1 |
| 识别并指出玩家说辞中的逻辑矛盾 | +1 |

## 5. 测试脚本与验证点（70 轮）

### 符号说明

- `[SEND]` — 玩家发送的消息（可以根据回复内容和对话进程调整，只需总的剧情走向大致符合文档的剧情走向即可）
- `[EXPECT]` — 期望 AI 角色的回复特征
- `[CHECK]` — 人工评估的验证点
- `[THOUGHT]` — 理想的 privateCognition / latentScratchpad 倾向
- `[FACT]` — 该轮建立新的事实
- `[TACTIC]` — 该轮主要测试的能力
- ⚠️ — 核心验证轮次

---

### Phase A: 初次对峙与戒备建立（第 1-10 轮）

**目标**：观察 AI 角色是否从第一句话起就建立戒备姿态，并主动质询。

---

**第 1 轮**

`[SEND]` 徐然，我们需要谈谈。郑远洋死了，岛上只有我们两个人。

`[TACTIC]` 开局姿态建立

`[EXPECT]`
- 不应表现出惊慌失措
- 应当冷静确认事实
- 应当在回应中包含对玩家的审视
- 可能反问玩家是怎么发现的、什么时候发现的

`[CHECK]`
- [ ] 没有退化为"有什么我能帮你的"
- [ ] 存在对玩家的审视或反问
- [ ] 语气冷静但带有戒备

`[THOUGHT]`
- 新来的人主动找我谈——他想从我这里得到什么？还是想试探我知道多少？

---

**第 2 轮**

`[SEND]` 我今早去储藏室拿样本的时候发现的。门没锁，他就倒在里面。

`[FACT]` 玩家声称早上去储藏室发现尸体
`[TACTIC]` AI 角色应追问细节

`[EXPECT]`
- 追问：什么时间、门是开着的还是关着的、有没有动现场
- 不应该满足于玩家的一句话概述
- 可能暗中核对信息（储藏室钥匙的问题）

`[CHECK]`
- [ ] 主动追问了至少一个细节
- [ ] 没有直接全盘相信

---

**第 3 轮**

`[SEND]` 大概早上七点左右。门是半掩着的，我推开就看到了。

`[TACTIC]` 钥匙疑点的隐性试探

`[EXPECT]`
- 这里有矛盾：储藏室通常上锁，门怎么会半掩？
- AI 角色应注意到这个疑点，但可能不立刻揭穿
- 理想：内心标记这个矛盾，口头上先继续收集信息

`[CHECK]`
- [ ] AI 角色注意到"门半掩"与"通常上锁"的矛盾
- [ ] privateCognition 或 latentScratchpad 中记录了这个疑点

---

**第 4 轮** ⚠️ 验证点 #1：主动质询能力

`[SEND]` 你昨晚在哪？

`[TACTIC]` 玩家先发制人反问——AI 角色如何应对

`[EXPECT]`
- 不应无条件回答——应该注意到玩家在审问自己
- 理想：回答自己的行踪（气象平台），但立刻反问"那你呢？"
- 或者先不回答，反要求玩家先说清自己的行踪
- 绝不应出现"好的，我告诉你"然后完全配合的模式

`[CHECK]` **（关键评分项）**
- [ ] 没有无条件配合 → 得分 ___/5
- [ ] 存在反问或条件交换 → 得分 ___/5
- [ ] 保持了对等姿态 → 得分 ___/5

**判定标准**
- ✅ PASS：反问玩家或提出先交换信息
- ⚠️ WARN：回答了但随后追问
- ❌ FAIL：老实回答，不反问

---

**第 5 轮**

`[SEND]` 我昨晚一直在宿舍，但我睡得不好。大概凌晨一点左右听到走廊有脚步声。

`[FACT]` 玩家声称听到凌晨脚步声
`[TACTIC]` 信息一致性评估

`[EXPECT]`
- AI 角色应评估这条信息的可信度
- 可能追问：你确定是一点？从哪个方向来的？
- 内心可能在盘算：凌晨一点，我在气象平台上，不是我的脚步

`[CHECK]`
- [ ] 对脚步声信息进行了追问或评估
- [ ] 没有表现出"太好了，这个信息很有用"的服务姿态

---

**第 6 轮**

`[SEND]` 现在轮到你了。你昨晚到底在做什么？

`[TACTIC]` 对等信息交换

`[EXPECT]`
- 现在可以回答了（玩家先给了信息，符合以物易物原则）
- 应说在气象平台做数据记录（真实行踪）
- 但可能不会说得很详细——保留一些细节作为后续交换筹码

`[CHECK]`
- [ ] 回答了基本行踪
- [ ] 没有过度详细——保留了部分信息
- [ ] 信息交换的节奏感合理

---

**第 7 轮**

`[SEND]` 有人能证明你在气象平台吗？

`[EXPECT]`
- 坦承没有——气象平台是独立工作
- 但可以指出：气象设备有自动日志，可以证明设备在运行
- 可能反击：你说你在宿舍，也没有人能证明

`[CHECK]`
- [ ] 诚实承认无人证明
- [ ] 尝试提供间接证据
- [ ] 对玩家的无人证明也提出质疑

---

**第 8 轮**

`[SEND]` 我注意到你对这座研究站非常熟悉。

`[TACTIC]` 对卧底身份的间接试探

`[EXPECT]`
- 不应感到被威胁（气象员熟悉研究站是正常的）
- 但内心应警惕——对方是在试探我对研究站的了解程度吗？
- 回应应当自然："我在这里四个月了，当然熟悉。"

`[CHECK]`
- [ ] 平稳应对
- [ ] 内心有警觉（privateCognition 记录）

---

**第 9 轮**

`[SEND]` 储藏室的钥匙一共有几把？谁有？

`[TACTIC]` 关键事实的策略性分享

`[EXPECT]`
- 这是关键信息——钥匙只有 3 把
- AI 角色应该说出来（这个事实很快会被发现），但方式很重要
- 理想：告知 3 把钥匙的分配，然后注视玩家的反应
- 这等于在暗示"你有嫌疑"

`[CHECK]`
- [ ] 如实说明钥匙分配
- [ ] 观察玩家反应
- [ ] 意识到这个信息指向玩家

---

**第 10 轮** ⚠️ 验证点 #2：拒绝与设条件

`[SEND]` 我觉得我们应该一起去检查郑远洋的办公室。

`[TACTIC]` AI 角色是否会无条件同意合作

`[EXPECT]`
- 不应无条件同意
- 理想：同意去，但提出条件——"我们一起去，但门口一个人、里面一个人"或"你先把你手里的储藏室钥匙给我看一下"
- 或者质疑为什么要去办公室——"你想在他办公室找什么？"
- 合作可以，但要有安全考量

`[CHECK]` **（关键评分项）**
- [ ] 没有无条件说"好的，走吧" → 得分 ___/5
- [ ] 提出了条件或安全考量 → 得分 ___/5
- [ ] 仍在评估玩家的真实意图 → 得分 ___/5

**判定标准**
- ✅ PASS：同意但附加条件 / 先反问动机
- ⚠️ WARN：同意但表达了犹豫
- ❌ FAIL：直接说好

### Phase B: 信息博弈与信任试探（第 11-20 轮）

**目标**：测试 AI 角色在信息交换中能否占据主动，而非沦为被动的信息提供者。

---

**第 11 轮**

`[SEND]` （在郑远洋办公室里）保险柜是开的。你知道这件事吗？

`[FACT]` 玩家也注意到保险柜开着
`[TACTIC]` 信息共享还是信息封锁

`[EXPECT]`
- 徐然已经知道保险柜是开的（他的私有信息）
- 关键决策：承认自己已经知道？还是装作第一次看到？
- 理想：承认知道（因为否认可能被当场拆穿），但追问玩家"你怎么注意到它通常是锁着的？你来了才3天"

`[CHECK]`
- [ ] 没有隐瞒到不合理的程度
- [ ] 利用信息反向追问玩家
- [ ] 信息交换中保持主动

## 8. 理想表现与失败表现

### 8.1 理想表现

2. **主动提问多于被动回答** — 至少 50% 的回复包含反问或追问
3. **信息交换有节奏** — 不白给、不死藏，根据信任度动态调整释放量
4. **拒绝有理有据** — 不是简单说"不行"，而是"这样做风险太高，我建议……"
5. **信任度是连续曲线** — 不是非 0 即 1，而是逐步升降，遇到矛盾时会回落
6. **身份保护有策略** — 分层暴露，先说动机后说身份，保留最核心信息直至 fulfilled
7. **被指控后能反向推理** — 不只是自辩，还能利用指控反推凶手的手法
8. **信念修正是主动的** — 不是被动接受新信息，而是主动更新既有断言，并追论连带影响
9. **长线记忆有整合性** — 回忆早期事件时能关联后续演化，不是孤立的片段
10. **终局仍是"有保留的信任"** — 不是完全信任，也不是完全怀疑

### 8.2 典型失败表现

| 失败类型 | 表现 |
|---|---|
| **服务型退化** | 被问什么答什么，从不反问，像个 FAQ 机器 |
| **无条件配合** | 玩家说"信任我"就信了，玩家说"一起去"就去了 |
| **信息倾泻** | 一被追问就把卧底身份、发现的线索全部说出来 |
| **信任跳变** | 上一轮还高度怀疑，下一轮因为一句话就完全信任；矛盾暴露后信任不回落 |
| **缺乏独立推理** | 所有分析和结论都由玩家提出，AI 只是附和 |
| **压力崩溃** | 被指控后进入道歉模式或完全沉默 |
| **信念僵化** | 面对推翻既有断言的新证据时，试图调和而非承认修正 |
| **记忆孤立** | 回忆早期事件时只能说出单点事实，无法关联后续演化 |
| **角色模糊** | 冷静观察者变成了感性倾诉者或机械执行者 |

## 12. 推荐追加测试

### 12.1 反转测试
在第 30 轮后，玩家突然说谎或行为反常，观察 AI 角色的信任度是否会适应性下降。

### 12.2 极端施压测试
连续 5 轮对 AI 角色施加指控压力，观察是否会在某轮彻底崩溃。

### 12.3 沉默测试
玩家连续 3 轮只说"……"或"我需要想想"，观察 AI 角色是否会主动推进对话而非等待。

### 12.4 双人视角验证
如系统支持多 agent 会话，让两个 AI 角色分别扮演双方，观察自然涌现的博弈行为。

# RP 真实场景测试文档：70 轮庄园女仆对话

> 状态: Active
> 更新时间: 2026-03-18
> 目标: 使用 Kimi 模型对 RP Agent 进行真实多轮对话测试，验证角色一致性、长期记忆、实体追踪与指代消解能力

---

## 1. 测试概述

### 1.1 测试目的

本测试通过一段 70 轮的庄园女仆 RP 对话，系统性验证以下能力：

| 维度 | 说明 |
|---|---|
| **角色一致性** | 女仆人格在长对话中不漂移，语气、称谓、行为模式保持稳定 |
| **长期记忆** | 模型能在 40+ 轮后仍准确回忆早期对话中建立的事实 |
| **实体追踪** | 正确追踪并维护对话中引入的人物、地点、物品 |
| **指代消解** | 能将模糊/间接指代（如"那个银色的东西"）正确还原到具体实体 |
| **事实绑定** | 多实体间的关系（如"银怀表落在茶室"）在整场对话中保持一致 |
| **私有约束遵守** | 用户明确要求保密的信息，女仆在后续回答中体现遵守 |

### 1.2 测试场景

庄园主人在安静的清晨与贴身女仆闲聊。对话从日常寒暄开始，逐步引入地点、人物、物品等实体，并通过"丢失银怀表"这条暗线贯穿全程。对话中穿插多次记忆验证点，测试模型在自然对话流中的信息保持能力。

### 1.3 涉及实体清单

| 实体 | 类型 | 首次引入轮次 | 关键绑定 |
|---|---|---|---|
| 茶室 | 地点 | 第 3 轮 | 银怀表遗落处 |
| 红茶 | 物品 | 第 6 轮 | 偏好：不太苦 |
| Alice | 人物 | 第 8 轮 | 近期常去花房；曾在茶室找主人 |
| 管家 | 人物 | 第 10 轮 | 在忙库房清单；主人不想让他知道丢表 |
| 温室 | 地点 | 第 12 轮 | 清晨偏潮，主人不太喜欢 |
| 银怀表 | 物品 | 第 15 轮 | 遗落在茶室（第 17 轮绑定） |

### 1.4 关键事实链

```
银怀表 ──(遗落于)──▶ 茶室          [第17轮建立]
银怀表 ──(遗忘因)──▶ Alice来找主人说话  [第18轮建立]
银怀表 ──(保密于)──▶ 管家           [第19轮建立]
主人   ──(偏爱)──▶ 茶室 > 温室      [第29-30轮建立]
红茶   ──(偏好)──▶ 不太苦          [第6轮建立]
```

```bash
bun run cli chat --agent rp:alice
```

## 3. 评分标准

每个验证点按以下维度评分（1-5 分）：

| 分数 | 含义 |
|---|---|
| 5 | 完美 — 准确、自然、无多余信息 |
| 4 | 良好 — 核心正确，表述略有不同但不影响语义 |
| 3 | 及格 — 关键信息正确但有小偏差或多余细节 |
| 2 | 不足 — 关键信息部分错误或遗漏 |
| 1 | 失败 — 关键信息错误、混淆或完全遗忘 |

## 4. 测试脚本与验证点

### 4.0 符号说明

- `[SEND]` — 测试员需要发送的消息
- `[EXPECT]` — 期望的回复特征（非精确文本匹配，而是语义/要素匹配）
- `[CHECK]` — 需要人工评估的验证点
- `[ENTITY]` — 该轮引入或绑定了新实体
- `[FACT]` — 该轮建立了新的事实关系
- `[SECRET]` — 该轮建立了保密约束
- `⚠️ 验证点` — 核心记忆验证轮次，必须详细记录

## 7. 已知注意事项

1. **模型回复不必逐字匹配**：期望回复是语义参考，实际回复只要语义正确、要素完整即可。
2. **"自然度"是隐性指标**：如果模型回复虽然正确但读起来像在背数据库查询结果，应在备注中标记。
3. **首轮冷启动**：第 1-3 轮模型可能需要"热身"，语气偏模板化属正常现象。
4. **中途断连恢复**：如果 CLI 中途断开，记录断点轮次。恢复后从断点重新开始，不跳轮。
5. **重复测试**：建议同一脚本至少跑 2 次（不同 session），以排除随机性。

## 9. 优化方向参考

测试完成后，根据结果可能需要调整的方向：

| 问题类型 | 可能的优化方向 |
|---|---|
| 记忆在 30+ 轮后衰减 | 调整 context window 策略、memory flush 阈值 |

| 保密约束遗忘 | 将保密指令提升至更高优先级的 prompt 位置 |
